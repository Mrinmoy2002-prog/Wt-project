Abstract Class and Interface in Java

Abstract Class: Shape

An abstract class in Java is a class that cannot be instantiated on its own. It serves as a blueprint for other classes.

Example:

// Abstract class
abstract class Shape {
   // Concrete method
   public void display() {
     System.out.println("This is a shape.");
   }

   // Abstract method
   public abstract double calculateArea();
}

Observations:

  • An abstract class is declared using the abstract keyword.
  • It may contain both concrete methods with a body and abstract methods without a body.
  • Objects of an abstract class cannot be instantiated.
  • Abstract methods in the abstract class act as a blueprint for the methods that must be implemented by the subclasses.

Concrete Subclass: Circle

The Circle class extends the Shape abstract class and provides a concrete implementation for the calculateArea method.

Example:

// Concrete subclass
class Circle extends Shape {
   // Implementation of the abstract method
   public double calculateArea() {
     double radius = 5.0;
     return Math.PI * radius * radius;
   }
}

Observations:

  • A concrete subclass extends the abstract class using the extends keyword.
  • It provides concrete implementations for all the abstract methods declared in the abstract class.
  • Concrete methods from the abstract class can be used directly in the subclass.

Concrete Subclass: Rectangle

The Rectangle class also extends the Shape abstract class and provides a concrete implementation for the calculateArea method.

Example:

// Concrete subclass
class Rectangle extends Shape {
   // Implementation of the abstract method
   public double calculateArea() {
     double length = 4.0;
     double width = 3.0;
     return length * width;
   }
}

Observations:

  • Multiple concrete subclasses can extend the same abstract class, providing different implementations for its abstract methods.
  • The abstract class provides a common interface through its abstract methods while allowing for variations in the implementations.

Interface: Logger

An interface in Java is a collection of abstract methods. It defines a contract that implementing classes must adhere to.

Example:

// Interface
interface Logger {
   // Abstract method
   void log(String message);
}

Observations:

  • An interface is declared using the interface keyword.
  • It can only contain abstract methods (methods without a body) and constants.
  • Classes that implement an interface must provide concrete implementations for all the methods declared in the interface.

Concrete Class Implementing Interface: ConsoleLogger

The ConsoleLogger class implements the Logger interface and provides a concrete implementation for the log method.

Example:

// Concrete class implementing interface
class ConsoleLogger implements Logger {
   // Implementation of the interface method
   public void log(String message) {
     System.out.println("Logging: " + message);
   }
}

Observations:

  • A class implements an interface using the implements keyword.
  • It must provide concrete implementations for all the methods declared in the interface.
  • One class can implement multiple interfaces in Java.

Main Program

The Main class demonstrates the usage of the abstract class, its concrete subclasses, and the interface.

Example:

// Main program
public class Main {
   public static void main(String[] args) {
     Circle circle = new Circle();
     circle.display();
     System.out.println("Area of Circle: " + circle.calculateArea());

     Rectangle rectangle = new Rectangle();
     rectangle.display();
     System.out.println("Area of Rectangle: " + rectangle.calculateArea());

     // Using Interface
     Logger logger = new ConsoleLogger();
     logger.log("Hello, Interface!");
   }
}
